# Plan 09-01: Rich Contextual Errors and Warnings

Goal: Expand error and warning types to include rich context like line numbers and input fragments.

## Research
- `ScraperError` is defined in `src/lib.rs`.
- `TemplateWarning` is defined in `src/lib.rs`.
- `FsmParser::parse` currently returns `Result<Vec<BTreeMap<String, Value>>, ScraperError>`.

## Strategy
1. Modify `ScraperError` to include a `DetailedParseError` struct with `line_idx`, `line_content`, and `message`.
2. Modify `TemplateWarning` to include `line_idx` where applicable.
3. Update `Template::parse_internal` in `src/engine/fsm.rs` to populate these details on error.
4. Move `calculate_coverage` from `tests/coverage.rs` to a new module `src/engine/coverage.rs` and integrate it.

## Tasks
1. [ ] Update `ScraperError` and `TemplateWarning` in `src/lib.rs`.
2. [ ] Create `src/engine/coverage.rs` and move `calculate_coverage` there.
3. [ ] Update `Template::parse_internal` to use the new `DetailedParseError`.
4. [ ] Implement a way to collect warnings during `parse` (e.g., return `Result<(Results, Warnings), Error>`).

## Verification

- Manual verification that parsing errors include line numbers.
