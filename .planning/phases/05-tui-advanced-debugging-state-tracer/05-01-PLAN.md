  <name>Task 1: Define TraceEvent model in debug.rs</name>
  <files>src/engine/debug.rs</files>
  <action>
Add TraceEvent and TraceEventType to the debug module to capture temporal FSM state:

```rust
#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
pub struct TraceEvent {
    pub line_idx: usize,
    pub state_before: String,
    pub state_after: String,
    pub variables: HashMap<String, serde_json::Value>,
    pub event_type: TraceEventType,
}

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
pub enum TraceEventType {
    LineProcessed,    // Every line processed
    StateChange,      // FSM transitioned to new state
    RecordEmitted,    // Record action executed
    RecordCleared,    // Clear action executed
}
```

Add `pub trace: Vec<TraceEvent>` field to DebugReport struct. Update `DebugReport::new()` to initialize trace as empty Vec.

Design notes from research:
- Store full variable snapshots per event (not delta encoding) for initial simplicity - add optimization only if profiling shows memory issues with real-world templates
- event_type enables filtering by stepping mode (line-by-line vs state-by-state vs action-by-action)
- variables is HashMap<String, serde_json::Value> matching engine's typed capture format

<task type="auto">
  <name>Task 2: Instrument FSM loop to record trace events</name>
  <files>src/engine/fsm.rs</files>
  <action>
In Template::debug_parse, extend the FSM loop to record TraceEvent at each line:

**At start of line processing loop** (before rule matching):
- Capture `state_before = current_state.clone()`

**After rule matching and action execution**:
- Determine event_type:
  - If state_before != current_state: TraceEventType::StateChange
  - If Record action executed: TraceEventType::RecordEmitted
  - If Clear action executed: TraceEventCleared
  - Otherwise: TraceEventType::LineProcessed
- Snapshot variables from RecordBuffer (use existing method or add `current_values() -> HashMap<String, serde_json::Value>` if needed)
- Push TraceEvent to debug_report.trace

**Handle EOF sentinel** (after main loop):
- If Record action is implicitly triggered by EOF (line_idx = lines.len()), record a TraceEvent with event_type: RecordEmitted

Implementation pattern from research:
```rust
let trace_event = TraceEvent {
    line_idx,
    state_before: state_before.clone(),
    state_after: current_state.clone(),
    variables: record_buffer.current_values(),
    event_type: /* determine from state change + action */,
};
debug_report.trace.push(trace_event);
```

Notes:
- RecordBuffer may need a helper method to extract current HashMap<String, serde_json::Value>
- Preserve Phase 4 decision: EOF records use line_idx = lines.len()
- Continue stacking from Phase 4 still applies; trace events record AFTER each match
