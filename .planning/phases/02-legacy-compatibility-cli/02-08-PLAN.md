  <name>Task 1: Enforce strict exit codes + error-format (including clap failures) and deterministic template resolution</name>
  <files>src/main.rs</files>
  <action>
    Implement the verifier-required adapter behavior in `src/main.rs`:
    - Pre-scan raw `std::env::args_os()` for `--error-format` (both `--error-format json` and `--error-format=json`) anywhere in argv.
      - Use the pre-scan selection ONLY to format clap/usage errors when clap parsing fails.
    - Switch to `Cli::try_parse()` and handle clap errors manually:
      - If the clap error kind is `DisplayHelp` or `DisplayVersion`: print clap's message (no JSON envelope) and `std::process::exit(0)`.
        - Be consistent about the stream (recommended: stdout for help/version).
      - Otherwise (real parse/usage errors): print to stderr in selected format and `std::process::exit(1)`.
    - Implement runtime `--error-format human|json`:
      - Human: a readable message to stderr.
      - JSON: emit a structured error envelope to stderr (must be valid JSON) and keep stdout empty.
    - Implement template spec resolution:
      - If `--template` points to an existing path: use it.
      - Otherwise treat it as an identifier and search CWD only for `<id>.textfsm|yaml|yml|toml` (restricted by `--template-format` when not `auto`).
      - 0 matches -> error; >1 matches -> ambiguity error with guidance.
    - Use the warning-returning template load API added in plan `02-07` and capture warnings for stderr.
  </action>
  <verify>
    1. `cargo test -q` passes.
    2. Help/version are success exits (no JSON envelope):
       - `cargo run -- --help 1>out.txt 2>err.txt ; test $? -eq 0` and `test -s out.txt`.
       - `cargo run -- --version 1>out.txt 2>err.txt ; test $? -eq 0` and `test -s out.txt`.
       - `cargo run -- parse --help 1>out.txt 2>err.txt ; test $? -eq 0` and `test -s out.txt`.
    3. `cargo run -- parse` exits `1` (missing required args).
    4. `cargo run -- --error-format json parse 2>err.json ; test $? -eq 1` and `python3 -c 'import json; d=json.load(open("err.json")); assert d.get("ok") is False'`.
    5. Order-independence: `cargo run -- parse --wat --error-format json 2>err.json ; test $? -eq 1` and `python3 -c 'import json; json.load(open("err.json"))'`.
  </verify>
  <done>All failures (clap or runtime) exit `1` and are formatted per `--error-format`; template identifier search is deterministic and ambiguity is an error.</done>
</task>

    - Resolve final input set deterministically:
      - Combine positional `inputs...`, repeatable `--input`, and expanded `--input-glob` matches.
      - Error if any glob pattern matches zero files.
      - De-dupe exact file paths and sort them deterministically.
      - Include stdin when `--stdin` is set.
      - Also include stdin by default when no explicit inputs were provided AND stdin is not a TTY.
      - Process file inputs first, stdin last.
      - Error if the final resolved input set is empty.
    - Transcript preprocessing warnings:
      - In `src/transcript/mod.rs`, strip ANSI escape sequences from the raw transcript bytes before any line splitting / prompt-echo cleanup.
      - When ANSI stripping removes any bytes, append a transcript warning that the CLI can surface to stderr.
      - Add a small regression test in `src/transcript/mod.rs` (module test) that feeds a string containing ANSI color codes and asserts:
        - returned blocks contain no `\x1b` bytes,
        - and at least one warning is emitted.
      - Update transcript preprocessing helpers so the CLI can get `(blocks, warnings)` and print warnings to stderr.
    - Parse pipeline:
      - Read each input source fully; preprocess; parse blocks.
      - Fail fast on first error.
      - Do not write any records to stdout until all sources succeed.
  </action>
  <verify>

    2. ANSI stripping is exercised by the new transcript module regression test.
