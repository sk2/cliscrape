Create `docs/template-authoring.md` as a recipe-driven guide (multiple small examples, not one giant tutorial) covering:

- Modern templates (YAML and TOML):
  - High-level structure (version, fields, and patterns vs states).
  - A minimal "single-state" recipe.
  - A multi-state recipe (brief) and when to use it.
  - Field types and how type conversion works (at record emission).
  - How to read schema errors (path-aware errors) and fix common issues.
- Legacy TextFSM templates:
  - Values, states, actions (Next/Continue/Record/Clear) and how they map to behavior.
  - Known semantic differences / unsupported actions that matter for migration (call out `Clearall` / `Error` as requiring manual review).
- Regex guidance (practical, not theory):
  - Safer patterns, anchoring, greediness pitfalls.
  - Avoiding catastrophic backtracking; mention timeouts and how to debug.

Embedded troubleshooting inside this guide:
- Use Live Lab as first step: `cliscrape debug -t <template> -i <input>` and what to look for.
- Understanding coverage/threshold warnings and how to iterate.
