<task type="auto">
  <name>Task 1: Add command-level spans and structured events in the CLI</name>
  <files>src/main.rs</files>
  <action>
In `src/main.rs`, add `tracing` spans/events around major command operations:

- `parse` command:
  - Emit an info event at start with fields: template spec, template_format, input_source_count (after resolution), strict, threshold, timeout.
  - Emit an info event at finish with fields: records, warnings, sources, elapsed_ms.
  - Convert the success status line (currently `eprintln!("Parsed ...")`) into a `tracing::info!` event gated by `--quiet` (when `quiet=true`, do not emit the success info event).
  - Convert template loader warnings and parse warnings currently printed via `eprintln!` into `tracing::warn!` events with fields: kind, line_idx (optional), message.

- `list-templates` and `show-template` commands:
  - Emit debug/info events for template enumeration/resolution and selected template name.

Keep stdout-only command outputs unchanged (`println!` that prints tables/JSON) so pipelines remain safe.
Avoid logging full template contents or full input text; only log identifiers and counts.
  </action>
  <verify>
Run:
- `cargo test`
- `RUST_LOG=cliscrape=info cargo run -- parse --template cisco_ios_show_version --stdin --format json < /dev/null 1> /tmp/out.json 2> /tmp/log.txt`
Confirm `/tmp/log.txt` contains structured lifecycle lines and `/tmp/out.json` remains valid JSON (may be empty).
  </verify>
  <done>
Parse/list/show commands emit structured events at boundaries; warnings/success status are routed through tracing and respect filtering.
  </done>
</task>
