<task type="auto">
  <name>Task 1: Add tracing dependencies and logging init module</name>
  <files>Cargo.toml, src/logging.rs</files>
  <action>
Add `tracing` and `tracing-subscriber` (with `env-filter` + `json` features) to `Cargo.toml` dependencies.

Create `src/logging.rs` with:
- `LogFormat` enum compatible with clap ValueEnum usage (Text/Json).
- `init_logging(verbosity: u8, format: LogFormat)` that:
  - Writes logs to stderr (`with_writer(std::io::stderr)`) and never stdout.
  - Uses `EnvFilter` from `RUST_LOG` when set; otherwise derives a default directive from `-v`.
  - Enforces precedence: `RUST_LOG` overrides `-v` (locked decision).
  - Uses `.try_init()` (not `.init()`) to avoid panics when something already set a global subscriber.
  - Maps verbosity defaults:
    - 0 => `warn` (keep warnings visible by default)
    - 1 => `cliscrape=info`
    - 2 => `cliscrape=debug`
    - 3 => `cliscrape=trace`
    - 4+ => `trace` (all targets)
  - For JSON format: use `tracing_subscriber::fmt().json()` (newline-delimited JSON) and include span context (`with_current_span`, `with_span_list`) with `flatten_event(true)`.

Do NOT initialize tracing from library code (`src/lib.rs`) or any library modules.
  </action>
  <verify>
Run `cargo test`.
  </verify>
  <done>
`cargo test` passes and `src/logging.rs` provides `init_logging` with stderr writer + EnvFilter precedence + JSON option.
  </done>
</task>
