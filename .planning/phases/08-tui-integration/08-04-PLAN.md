# Plan 08-04: Integration with Live Lab and Selection

Goal: Support selection and loading of templates into the editor/parser.

## Research
- `src/tui/mod.rs` handles template selection from the `Picker` already.
- `AppState` manages `template_path`.

## Strategy
1. Add `Enter` key handling in `handle_key_browser` to select a template.
2. If an embedded template is selected, it must be written to a temporary file (or the editor needs to handle in-memory content).
3. Update `AppState::template_path` (and optionally `input_path` if the user is in picker mode).
4. Reload the parser and watcher with the new template.

## Tasks
1. [ ] Implement `TemplateBrowserState::select_current` to return the template path (or in-memory content).
2. [ ] Add `handle_key_template_browser` in `src/tui/mod.rs`.
3. [ ] If an embedded template is selected, write it to a temporary location (or update the editor to support in-memory templates).
4. [ ] Trigger a re-parse and re-watch with the new template.

## Verification
- Selecting a template in the browser loads it into the editor and parser.
- Parser uses the selected template correctly.
- If a user-placed template is selected, it loads without a temporary file.
- `q` or `Esc` cancels the selection.
